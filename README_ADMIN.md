# TournamentTimer Admin Guide

This guide is for the person running a TournamentTimer event: starting the server, opening the admin panel, preparing run assets, sharing runner credentials, and adding overlays or camera sources to OBS.

TournamentTimer is not a replacement for LiveSplit or OBS. It is the tournament layer: server state, admin control, runner sync, logs, audit data, and OBS-friendly browser sources.

## Basic workflow

TournamentTimer works like this:

```text
Runner UI / LiveSplit / ASL Host
-> TournamentTimer Server
-> Admin panel
-> OBS overlay / camera sources
```

The server stores the official run state. Runner clients send Start and Split events. The admin panel monitors the race and can apply emergency corrections. OBS displays browser sources generated by the server.

Before a real race starts, the admin should create a clean attempt with New attempt.

## What to prepare

Before a test or event, prepare:

```text
timer-settings.json
configs/<runId>.json
server-runs/assets/<runId>/splits.lss
server-runs/assets/<runId>/autosplitter.asl     optional
adminKey
runKey
viewKey
server URL
```

Usually you only need to edit:

```text
timer-settings.json
configs/<runId>.json
```

If you use LiveSplit splits, the server can load split names from `splits.lss`.

## Main files and folders

Typical structure:

```text
TournamentTimer/
  timer-settings.json
  timer-settings.example.json

  configs/
    local-test-run.json
    re9.json

  scripts/
    start-server.ps1
    open-admin.ps1
    start-runner-ui.ps1

  server-runs/
    assets/
      re9/
        splits.lss
        autosplitter.asl
        asl-settings.json

  local-runs/
```

`timer-settings.json` is local runtime configuration. Do not commit real keys to a public repository.

Example:

```json
{
  "runId": "re9",
  "serverUrl": "http://localhost:5177",
  "runConfigPath": "configs/re9.json",
  "adminKey": "long-random-admin-key",
  "runKey": "long-random-run-key",
  "viewKey": "long-random-view-key"
}
```

Important notes:

- the server reads `timer-settings.json` at startup;
- if you change `runId`, keys, `serverUrl`, or `runConfigPath`, restart the server;
- use forward slashes in `runConfigPath`, for example `configs/re9.json`;
- `timer-settings.example.json` is only a template;
- real keys should stay private.

## Access keys

TournamentTimer uses simple access keys. They are not full user accounts or roles. They are a practical barrier for small events and test deployments.

```text
adminKey
  Used by the admin panel and dangerous actions:
  New attempt, Reset, Admin Start, Admin Split, Undo, Finish.

runKey
  Used by Runner UI clients to connect and send events.

viewKey
  Used by overlays and read-only browser sources.
```

Give `adminKey` only to the admin. Give `runKey` only to runners. Give `viewKey` to OBS or read-only viewers.

## Run ID

Run ID is the technical name of the run. It is not necessarily the display title.

Example:

```text
Run ID: re9
Game: Resident Evil 9
Category: Any%
```

The same Run ID should match in:

```text
timer-settings.json
configs/<runId>.json
server-runs/assets/<runId>
Admin panel Run ID field
Runner UI Run ID field
```

If one place says `re9` and another says `local-test-run`, clients will not connect to the correct run.

## Prepare a run config

Do not write a config from scratch unless needed. Copy an existing template:

```text
configs/local-test-run.json
```

For example:

```text
configs/re9.json
```

Edit at least:

```json
{
  "runId": "re9",
  "game": "Resident Evil 9",
  "category": "Any%"
}
```

Then check `timer-settings.json`:

```json
{
  "runId": "re9",
  "runConfigPath": "configs/re9.json"
}
```

## Add LiveSplit splits

For Run ID `re9`, create:

```text
server-runs/assets/re9/
```

Put the LiveSplit splits file there:

```text
server-runs/assets/re9/splits.lss
```

Recommended rule: keep one `.lss` file in a run assets folder. If there are several files, name the main one `splits.lss`.

If `splits.lss` exists, the server uses it as the split source.

## Add an autosplitter

If the run needs an autosplitter, put it in the same assets folder:

```text
server-runs/assets/re9/autosplitter.asl
```

Supported asset types can include:

```text
.asl
.wasm
.dll
```

Support depends on the runtime path being used. Normal ASL files are handled by ASL Host tooling. WASM / AutoSplittingRuntime is currently a separate future layer, not a normal ASL file.

If no autosplitter is needed, leave it empty. The admin panel may show Autosplitter: none. That is fine.

## Start the server on Windows

Open the TournamentTimer folder and run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-server.ps1
```

Or right-click `scripts/start-server.ps1` and choose Run with PowerShell.

Keep the server window open.

The server should print something like:

```text
Starting TournamentTimer.Server
RunId: re9
ACCESS KEYS: admin=on, run=on, view=on
```

If the server reports a runId mismatch, check `timer-settings.json` and the selected config file.

## Start the server on Linux

Use the server-admin Linux package.

Example:

```bash
cd /opt/tournamenttimer
chmod +x ./server/TournamentTimer.Server
chmod +x ./scripts/start-server-linux.sh
./scripts/start-server-linux.sh
```

For a public or VPN server, `serverUrl` usually needs to bind to all interfaces:

```json
{
  "serverUrl": "http://0.0.0.0:5177"
}
```

Open TCP port 5177 in the server firewall and any hosting security group.

If using systemd, use the provided service example as a template and adjust paths for your server.

## Open the admin panel

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\open-admin.ps1
```

The script reads `timer-settings.json` and opens the admin URL with the correct keys.

Manual local URL example:

```text
http://localhost:5177/admin.html?runId=re9&adminKey=...&runKey=...&viewKey=...
```

Normally, do not build this URL by hand.

## Admin panel overview

The admin panel shows:

- current run and attempt;
- connected runners;
- runner status;
- current split;
- split table;
- final results;
- client time;
- server time;
- delta;
- overlay URLs;
- per-runner overlay URLs;
- camera source URLs;
- runner camera status;
- server and runner launch commands.

## LiveSplit input lock warning

If a runner card is highlighted in red and shows `LIVE SPLIT INPUT LOCKED`, Runner UI rejected LiveSplit input for that attempt.

Common reasons:

```text
invalid LiveSplit GameTime
LiveSplit reset/undo/skip during a running attempt
local/server desync
wrong attempt after New attempt
```

For LRT/load-removed runs, invalid GameTime means the official LiveSplit split was not accepted. The bad split is not written as an accepted official server event. The admin should decide outside the automatic timing path: new attempt, technical ruling, RTA fallback by event rules, or manual note of a time adjustment.

Do not treat a red LiveSplit input lock as a normal runner state. It means something went wrong and the runner's LiveSplit input is no longer trusted for the current attempt.

## Client time, server time, and delta

TournamentTimer keeps timing audit data.

```text
Client time
  Time reported by Runner UI or LiveSplit.
  This is normally the official event time.

Server time
  Control time from the server side.

Delta
  Difference between server time and client time.
```

Client time is the primary timing source. Server time is used for audit and control.

Example: if a runner splits at `10:00.000`, but the event reaches the server five seconds later, the accepted split time is still `10:00.000`.

Large delta does not automatically mean a different result. It means the logs should be checked.

Useful logs:

```text
server-runs/*.server.events.jsonl
server-runs/*.admin.events.jsonl
server-runs/*.server.snapshots.jsonl
local-runs/*.events.jsonl
local-runs/*.server-sync.jsonl
```

## OBS overlay

Use overlay links from the admin panel.

There are two common overlay types:

```text
shared overlay
per-runner overlay
```

In OBS:

1. Add Browser Source.
2. Paste the overlay URL from the admin panel.
3. Set the source size.
4. Crop and position it in OBS.

If `viewKey` is enabled, overlay URLs need the view key.

If the URL starts with `localhost`, it only works on the same machine. For another PC, open the admin panel through the real server IP, VPN IP, or domain and copy the URL again.

## Runner cameras

Runner cameras are optional. Runner UI can publish camera video for OBS browser sources.

Runner camera flow:

```text
Runner UI camera
-> TournamentTimer Server signaling
-> admin preview / OBS camera source
```

Runner UI captures camera only. It does not handle microphone audio. Voice chat should stay in Discord or another voice service.

In the admin panel, use the Camera sources section:

- check runner camera status;
- open preview for quick checks;
- copy the OBS camera source URL.

In OBS:

1. Add Browser Source.
2. Paste the camera source URL.
3. Use a source size such as 1280x720.
4. Crop and position in OBS.

For remote events, camera reliability depends on network conditions. HTTPS and TURN may be needed for difficult NAT, VPN, or mobile network cases.

## TURN notes

TournamentTimer can read ICE/TURN settings from `timer-settings.json` and provide them to camera pages.

Example:

```json
{
  "CameraIceTransportPolicy": "all",
  "CameraIceServers": [
    {
      "Urls": [
        "stun:stun.l.google.com:19302"
      ]
    },
    {
      "Urls": [
        "turn:turn.example.com:3478?transport=udp",
        "turn:turn.example.com:3478?transport=tcp",
        "turns:turn.example.com:5349?transport=tcp"
      ],
      "Username": "CHANGE_TURN_USERNAME",
      "Credential": "CHANGE_TURN_PASSWORD"
    }
  ]
}
```

Transport policy:

```text
all
  Normal mode. WebRTC tries direct/STUN and falls back to TURN.

relay
  Test mode. Forces relay-only TURN usage.
```

To check server-side camera configuration:

```text
/api/server-info?key=ADMIN_KEY
/api/runs/<runId>/media/ice-servers?key=VIEW_KEY
```

TURN itself must be configured on a VPS or external service. TournamentTimer only consumes the TURN configuration.

## Admin actions

Admin actions are for corrections and emergency control.

```text
Admin Start
  Start a selected runner from the admin panel.

Admin Split
  Apply the next split for a selected runner.

Undo
  Undo the last split or finish.

Finish
  Finish a selected runner manually.

New attempt
  Create a clean attempt.
  Use this before the real race after tests.

Reset server only
  Rebuild the current attempt state from server logs.
  This does not create a new attempt.
```

Normal runner finish should happen through the final split or LiveSplit bridge. Manual Finish is an admin fallback.

## Before the real start

Recommended checklist:

1. Check `timer-settings.json`.
2. Start the server.
3. Open the admin panel.
4. Check server settings and access keys.
5. Check run config and split source.
6. Add overlay sources to OBS.
7. Add camera sources to OBS if used.
8. Connect all runners.
9. Confirm each runner has a unique Runner ID.
10. Ask runners to start cameras if used.
11. Do a test Start/Split.
12. Check split table, client time, server time, and delta.
13. Check overlays and cameras in OBS.
14. Press New attempt.
15. Start the real race.

## Common problems

### Server does not start: runId mismatch

Check:

```text
timer-settings.json runId
config file runId
timer-settings.json runConfigPath
```

Restart the server after fixing settings.

### Admin panel shows 401

Usually wrong or missing key.

Check:

```text
adminKey
viewKey
URL opened through open-admin.ps1
```

### Admin panel shows 404

Usually wrong Run ID or the server was started with a different config.

### Runner cannot connect

Check:

```text
Server URL
Run ID
Runner ID
Run key
server is running
firewall allows TCP 5177
VPN is connected if used
```

### Splits are wrong

Check:

```text
server-runs/assets/<runId>/
splits.lss
config splits fallback
```

If there are several `.lss` files, name the intended one `splits.lss`.

### Need a clean start

Press New attempt in the admin panel.

After New attempt, old Runner UI windows may need to reconnect to the new attempt.

### Camera is offline

Check:

```text
runner clicked Connect
runner clicked Start camera
camera permissions in Windows
viewKey in camera source URL
admin panel opened through real server address
```

### OBS does not show camera

Check:

```text
camera source URL copied from admin panel
URL does not use localhost unless OBS is on the same machine
viewKey is included if keys are enabled
Browser Source was refreshed
network/VPN/WebRTC restrictions
TURN configuration if needed
```
